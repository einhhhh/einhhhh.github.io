The summary of TCP/IP Illustrated Volume 1:chapter two

发表于 2019-10-02 更新于 2019-10-03 Valine：

Introduction

This chapter is quite easy since it only mentions a few of the common data-link technologies. I think there will be more complex things about the Link Layer that will be mentioned in the following chapters or it will be too easy for the readers to understand the Link Layer. Ok, this is my feeling after I finished this chapter. Now, let’s start the major part of this blog.
The chapter two involve the details of some important technologies in Link Layer which include Ethernet and IEEE 802 Encapsulation,SLIP and Compressed SLIP(the acronym SLIP stands for Serial Line IP),the PPP which is Point-to-point Protocol for short,the Lookback interface,the MTU and Path MTU AND the Serial Line Throughput Calculations.
In the rest part of this blog, I will write a summary of those concepts which I think is the most important.

Ethernet and IEEE 802 Encapsulation

So, as we know, when data goes down in the Protocol Stack, the Encapsulation will happen. (In my last blog about chapter one, I mention the concept of Encapsulation. If you don’t know what that means, you can read my last blog.)And the Ethernet and IEEE 802 Encapsulation is such an action that happens in the Link Layer.
There are 2 RFCs you need to read to understand more details about those two encapsulations: the RFC 894 which for Ethernet and RFC 1042 which for IEEE 802 network. And the format of these Encapsulations are below(btw, sorry for such a poor quality picture…).

SLIP,Compressed SLIP and PPP

In my opinion, the relationship between these three technologies is that the later one is the former ones upgraded version. And I am gonna summarize the three technologies one by one.

1.SLIP

We need to know that, the SLIP, CSLIP, and PPP are all the forms of encapsulation on the serial line which is in the Linke Layer.
About the SLIP, you can find more details in the RFC 1055. As a result, I will just mention some shortcomings of this tech in this blog. The format of this Encapsulation is below:

The SLIP has three shortcomings that are worth noting:

  1. When delivering data, each end must know the other’s IP address;
  2. No type field in the frame. Because of this, It can’t be used for some other protocol at the same time when the serial line is used for SLIP;
  3. No chechksum(This term is something I didn’t mention in the Ethernet and IEEE 802 Encapsulation part. It is related to the CRC field at the end of the frame and It is a cyclic redundancy check that detects errors in the rest of the fame.).That means it is the higher layer’s responsibility to detect the datagram’s corruption. Which is also means, it is essential that the higher layer must provide some form of CRC.BTW, there is always a checksum for IP header, TCP header, and TCP data while the checksum is optional in UDP header and UDP data.
    2.Compressed SLIP
    Since SLIP lines are often slow, the scientist develops a newer SLIP called CSLIP which stands for the Compressed SLIP. The CSLIP normally reduces the 40-bytes header to 3 or 5 bytes and these smaller headers greatly improve the interactive response time.
    Faster and more effective, that’s why I call it the upgraded version of SLIP.
    3.Point-to-Point Protocol
    The PPP corrects all the deficiencies in SLIP and you can read the RFC 1548 and RFC 1332 to find more details about this tech. The format of the PPP frame is below:

    One thing I think we should note is the flag field in the PPP frame. The flag is in the begin and the end of the frame and the value of this flag is 0x7e. The flag value can not be changed, so the PPP needs to ESCAPE this byte when it appears in the information field.the technology to escape this byte called bit stuffing.
    Compared to the SLIP, the PPP has the following advantages:
  4. Support for multiple protocols on a single serial line;
  5. Cyclic redundancy check on every frame;
  6. Dynamic negotiation of IP addresses for each end;
  7. TCP and IP header are compressed similar to CSLIP;
  8. A link control protocol for negotiating many data-link options.

MTU and Path MTU

There is a limit on the size of the frame for both Ethernet and IEEE 802 Encapsulation. This characteristic of the Link Layer is called the MTU, it is the Maximum Transmission Unit.
If IP has a datagram to send and the size of the datagram is larger than the MTU, the datagram will be broken into smaller pieces. This action called fragmentation. The following figure is the Typical MTUs.

When the two hosts are communicating in the same network,it is easy for both of them to find the MTU.But how the two hosts to know the MTU when they are in different network?That’s what the Path MTU to do.
The Path MTU will help two hosts in different network to find thier MTU by finding the smallest size of frame can be deliver between the two host.You can see more detalis in RFC 1191.And more details also will be samarized in the following sammarys.
