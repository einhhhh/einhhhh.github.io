Introduction

It took me about 3 days to finish reading chapter 3 which make a short introduction to IP, the Internet Protocol. This chapter mentions IP Header, IP Routing, Subnet and two useful commands. Before we start to summarize these things, let’s see the overview of the IP.
IP is the workhorse protocol fo the TCP/IP protocol suite. An important thing we should know is that IP provides an unreliable, connectionless datagram delivery service. The term “unreliable” means there are no guarantees when an IP datagram successfully gets its destination and the term “connectionless” means that IP does not maintain any state information about successive datagrams which also means the IP datagrams can get delivered out of order.
You can see more details about IP in RFC 791. Now, let’s begin writing our summary.

The first thing we should know is that the 4 bytes in the 32-bits value are transmitted in the order: bits 0-7, bits 8-15, then 16-23,and 24-31 last. This is called big-endian byte ordering which also called network byte order.
