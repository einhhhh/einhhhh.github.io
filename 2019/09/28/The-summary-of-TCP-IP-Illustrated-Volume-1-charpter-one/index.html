A Short Introduction

Hello,the TCP/IP Illustrated is the first english textbook I decided to read.To help me understand those concepts, I will write a series of blog about my study.But I will not write those concepts in detail,these blogs will just something like notes or some thoughts of me.And This is my first time to write a techblog in Einglish,so if there have something wrong,please tell me the wrong places.Thank you!Ok,let’s begin!

About this Chapter

This chapter is the overview of the TCP/IP Protocol Suite.Just like the author said,”This chapter has been a whirlwind tour …”It introduces following parts and some important concepts in TCP/IP:

1.The Layer of TCP/IP Protocol Suite

The TCP/IP Protocol Suite includes four layers:Application,Transpot,Network and link.Each layer has different use,and the details of each layers will be introduces in the others chapters.

2.Internet Addresses and DNS(Domain Name System)

Every interface on an internet have a unique Internet address.These addresses are 32-bit numbers and these 32-bit addresses are written as fout decimal numbers which we call it dotted-decimal address.There are five classes of address.We can tell them apart by looking at the first number of those addresses.For example,If an address begin with only 0,It is a Class A.I am gonna to pose a picture about the differences between different classes of internet address after I know how to do it. :)
As we know,It is easier for us to remenber a sentence rather than a string of numbers.As a result, the scientists develop the DNS which is Domain Name System for short.The Domain is the name of a host.The DNS is a distributed database that provides the mapping between IP addresses and hostnames.

3.Encapsulation and Demultiplexing

This term, Encapsulation, is refer to an action that would happen when the data goes down the procotol stack.When the data is deliver from one layer to the other layer,a header will be add on the head of the date.The header includes some basic information of the former layer.And as you can see, this action is called encapsulation.
As the contrary part,the Demultiplexing is an action happened when the data is recived at the destination host it starts its way up the protocol stack and the headers will be removed by the appropriate protocol box.

4.Other Concepts

The chapter one also mention some concepts like Client-Server Model,Port Numbers,RFCs and so on.
1.About the Client-Server Model
Most networking applicaions are written assuming one side is Client and the other side is Serve.And what we need to know is that,There are 2 kinds of Server,the iterative Server and the concurrent Server.
2.About the Port Numbers
Servers are normally knom by their well-known porst number.Those services that can be provided by any implementation of TCP/IP have well-known port numbers between 1 and 1023.
Client Port numbers called ephmeral ports,most TCP/IP implementation allocate ephmeral ports numbers between 1024 and 5000.
The port numbers above 5000 are intended for other servers.
